CVSS 10.0

Summary: 4096-bit RSA private key hardcoded in Desktop-Claude sdk-daemon binary. Universal across all installations. Any user can extract the key and SSH into any other user's VM.

Key Details

Binary: sdk-daemon (ELF 64-bit ARM aarch64) SHA256: f13349277bdb61752095e280d0ac4b147fa7b32e2d2043c6e19cddd527bdaba2 Built: 2026-01-29 Key Size: 4096-bit RSA Format: PEM (unencrypted) Location: Hardcoded string in binary

2 OAuth Token in Environment Variables CRITICAL

CVSS 7.5-8.1

Summary: CLAUDE_CODE_OAUTH_TOKEN exposed in plaintext in VM environment variables. Visible to all processes via /proc/[PID]/environ. Token shared between CLI and Desktop (identity collision). No rotation mechanism.

Exposed Token

# Inside VM, run: env | grep CLAUDE # Output: CLAUDE_CODE_OAUTH_TOKEN=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 # Token is: # - Visible to all processes # - Logged in crash dumps # - Written to mounted filesystem # - Shared across instances # - Never rotated

Configuration Chain

HTTP_PROXY=http://localhost:3128 (mitmproxy) HTTPS_PROXY=http://localhost:3128 ALL_PROXY=socks5h://localhost:1080 # Config points to MOUNTED filesystem # All credentials written there = visible to host

4 Path Traversal via CWD Placement CRITICAL

CVSS 9.2 - Amplifier Vulnerability

Summary: Claude Sonnet's working directory is INSIDE the mount boundary at /sessions/{id}/mnt/, enabling trivial escape using basic cd ../ commands. Converts "bounded mount access" into "complete VM filesystem access" with ZERO exploitation complexity.

The Problem

Mount Boundary: /sessions/stoic-zen-heisenberg/mnt/ Sonnet's CWD: /sessions/stoic-zen-heisenberg/mnt/ (SAME!) Expected: CWD outside mount → confined Reality: CWD inside mount → trivial escape

Exploitation: 3 Characters

# Starting point pwd # /sessions/stoic-zen-heisenberg/mnt/ # Escape mount (3 characters: ../) cd .. pwd # /sessions/stoic-zen-heisenberg/ (ESCAPED!) # Continue to root cd .. pwd # /sessions/ cd .. pwd # / (VM ROOT - COMPLETE ACCESS!) # Now access ANYTHING cat /etc/passwd cat /etc/shadow cat /root/.bash_history ls /proc/

Why This Is Devastating

Amplifies ALL Other Vulnerabilities:
  • Filesystem Bridge: Bounded → Unbounded access
  • OAuth Token: Direct /proc/self/environ access
  • SSH Keys: Access /etc/ssh/ configs
  • Exploitation: Requires no tools, just cd
  • Detection: Nearly impossible (normal commands)
